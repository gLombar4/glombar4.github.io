The Four Pillars

Within the concept of Object-Oriented Programming are the indivial concepts known as "The Four Pillars".

These Pillars are:

  • Abstraction
  • Encapsulation
  • Inheritance
  • Polymorphism

Abstraction

Abstraction is the process of hiding implementation details & allowing a developer to work with only essential components. For example, using a function to handle specific operations that the developer shouldn't be concerned about.

In this example, we create a function to print a hello message, then we can just use that function later instead of having to perform whatever operation that function contains again.

Encapsulation

If you've seen the "Getting Started" section you've already seen an example of this! Encapsulation essentially means containing things relating to a class inside of that class, instance variables, methods, etc.

Inheritance

Inheritance involves organizing classes into "super" and "sub" classes. For example, having an "Animal" superclass, of which "Dog" and "Cat" are subclasses.

Here we've defined an "Animal" class & have given all Animal objects a name attribute.

Now we'll make a Dog class and define it as a subclass of Animal. (Note the Extends keyword.) super(name) makes it so that all Dogs have names, but only Dog objects will have the "breed" attribute we've specified as part of this class. We've just made a superclass-subclass relationship.

Polymorphism

Polymorphism means being able to use the same piece of code on multiple types of objects. From the above "Animal", "Cat" and "Dog" example, you should be able to use the same print method to print out information about cats and dogs. Despite being different objects & being made from different classes, they are both part of the Animal class, & thus can use polymorphism to their advantage.

Building off the previous inhertiance example, notice how getName() is from the Animal class. Despite this, since Dog and Cat objects are also Animals, they can use this method.
